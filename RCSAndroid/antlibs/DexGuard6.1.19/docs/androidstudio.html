Android Studio

DexGuard can be run from Android Studio (currently tested with version 1.2). Android Studio internally performs its builds with Gradle, so you simply need to apply DexGuard's Gradle plugin.

Setting up your project

Android Studio creates a standard build.gradle file in your project. You can edit it to apply the DexGuard Gradle plugin, as documented in the Gradle Plugin section.

Building

The procedure for building Android applications and libraries remains the same. You can use the standard ways in Android Studio to start your debug builds and release builds.

Further tips

The same tips for Gradle builds apply here. Notably:
  • Gradle by default swallows the logging output of its tasks, including the DexGuard task. If you want to get verbose output for the entire build process, you can run Gradle with a more verbose log level by adding the options --info, --debug, or --stacktrace in the Gradle properties panel:
    File > Settings... > Build, Execution, Deployment > Compiler > Compiler > Command-line Options
    You then also see DexGuard's log output, which can be made more informative with the option -verbose in your dexguard-project.txt file.

Reverting to the traditional Android build

You can go back to the basic Gradle build of the Android SDK by removing the changes from the build.gradle file of your project.
